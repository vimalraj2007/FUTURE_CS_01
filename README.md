# 🔐 Vulnerability Assessment – demo.testfire.net

## 📌 Overview

This project presents a basic security assessment of a public website using safe, read-only methods.

The goal was to identify common security weaknesses and suggest simple improvements.

---

## 🌐 Website Tested

* http://demo.testfire.net/

---

## 🎯 Scope

* Public pages only
* No exploitation
* Passive analysis

---

## 🔍 What Was Checked

* Open ports and services
* Security protections in the browser
* Cookie and session handling
* Visible system behavior

---

## ⚠️ Key Findings

* Missing browser security protections
* Weak session (cookie) security
* Extra exposed services
* Limited protection against unauthorized actions

---

## 📊 Risk Level

**Overall: 🔴 High**

* High → Serious issues needing immediate fix
* Medium → Should be improved soon
* Low → Minor issues

---

## 🛠 Recommendations

* Use secure connections (HTTPS)
* Improve browser security settings
* Strengthen session protection
* Reduce unnecessary exposure
